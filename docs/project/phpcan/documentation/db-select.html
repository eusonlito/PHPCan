Db::select()

phpCan 1.1

Description

Execute a selection in database and return the result.

Db::select (string/array $operations)

operations (array/string)
Array with all params for the selection. If you only want specify the table name, it can be a string.

How does it work?

This function execute one or more selects in the database and return the results. You only have to specify the params you want and it detects automatically the relations between tables to build the query and execute it.

There are many params you can pass:

table
Name of the table where you want make the selection.
fields
Array with the fields you want to select. You can also put "*" or "**", or if you want to select just one fields, it can be a string.
fields_commands
An string or an array of literal commands you want use as fields. For example "COUNT(*)".
conditions
An array with conditions you want the resulted rows met.
conditions_or
If you want the rows met only one of this conditions.
conditions_and
If you want the rows met all conditions. Its the same of "conditions" param.
limit
A number limiting the selected rows.
offset
The position where start to select rows.
rows
If the limit is 1, by default, the function returns an array with the values of the selected row. If you want get the result in the same way of a multirows selection, set this param as true.
exit_format
The format of the retrieved data. It can be "html", "form" or false. By default is "html".
sort
String with the name of the field you what use to sort the results. If you want use more than one, you have to use an array.
sort_direction
Direction of the sort. It can be ASC or DESC. You can also put the direction just in sort param (for example "id DESC")
sort_commands
A string or an array of literal commands for sort the result. For example "RAND()" for sql.
group
A string or an array with the name fields you want use to group the result.
paginate
An array with the info to generate a pagination of the results.
add_tables
Tables added to this selection. These tables will generate additional selections.
join_tables
Tables added to this selection. These tables won't generate additional selections, they will be joined to the main selection.

Do not worry about relations

Yes, phpCan know the relations of all tables, so it can be generate the conditions and queries necessary to join them.

For example, if you want get all comments of the post with id = 2, you only have to say "give me the comments where post.id = 2".

What is add_tables?

Add_tables says to the function that you don't want only the main table, but you want data from other related tables.

For example, if you want the data of the ten latest posts and all their comments, you don't need to put the table comments in "add_tables", and the function will select the posts, and the comments of each post and will return an array with all correctly ordered data.

The params of add_tables are the same of the main selection: table, fields, limit, sort, etc, and others add_tables. So you can select the posts, add the table comments and add the table users to comments. The function will execute all queries recursively.

What is the difference between add_tables and join_tables?

Add_tables generate extra queries. For example for select the latest ten posts with their comments, the function first select the latest ten posts, get their ten ids and then select the comments related with these ids. Finally put each comment in a subarray of its post.

Join_tables select the extra tables in the same operations. It works like "left join" in MySql, but the resulted rows will be ordered in a subarray, like add_tables. As "left join", it can be generate duplicate rows, so is recommended only for certain cases.

How works the paginate param?

Paginate is useful if you want make the selection in a paginated page. You have to define the current page (if the page is 0, it became to 1) and the function will use the param limit to know how many rows have each page.

You can also add the param "map" to generate a map of previous and next pages (Google style). The paginate info will be available in the variable $pagination (you can change the variable in the param store). The $pagination variable is an array which contains:

page
Number of current page.
previous
Number of the previous page. If the current page is 1, the previous page is false.
next
Number of the next page. If we are in the last page, the next page is false.
total_pages
Number of total pages.
first
First page in the map. If we've defined the map in 5 and we are in page 8, the first page is 5.
last
Last page in the map. If we've defined the map in 5 and we are in page 8, the last map would be 13 if the total pages isn't lower than 13.
total
Number of total rows selected.

How conditions work?

To define conditions you can use the params "conditions", "conditions_and" and "conditions_or", but all params are the same because you can put "conditions_or" inside a "conditions_and" and viceversa. "Conditions" and "conditions_and" are the same.

The keys of the array are reserved for the tables, fields and operator and the values for the value you want to check. For example, if you want to check if the id is lower than 3, the array would be array('id <' => 3).

You can define conditions using one or more tables. For example, if you want to check if the user of the comment of the post has id 4, you have to write: array('posts.comments.users' => 4).

You can define literal conditions, like the function Database::select. To do that, you have to write all condition in the value. For example array('`coments`.id = 3');

You can define subcondition (conditions inside conditions). If the parent condition is type "and", the subcondition is "or" and viceversa. A subcondition is an array of conditions inside another array of conditions.

Some relations have to have a name. For example, if you want relate the posts with users to get the authors of the posts, and you want relate again the posts with users to get the users who like each post, you have to relate two times users with posts, so to identificate each relation you have to take them names. For example "authors" and "likes". If you want select the posts by authors, you have to put the name or relation in parenthesis: array('posts.users(authors).id' => 4);

Other relations can have a direction, for example if you want relate the categories with itselves to get subcategories. To get the categories related with other category you have to put the direction with a "-". For example to select the childs of the category with id 4, you have to write: array('categories.categories-parent.id' => 4).

//Select all categories
$categories = $Db->select('categories');

//Select the latest 10 posts with their comments
$posts = $Db->select(array(
	'table' => 'posts',
	'limit' => 10,
	'sort' => 'id DESC',
	'add_tables' => array('comments')
));

//Select the latest 10 active posts of the user "john" and with its validated comments and authors of the comments
//Select also the tags name of the posts
$posts = $Db->select(array(
	'table' => 'posts',
	'limit' => 10,
	'sort' => 'id DESC'
	'conditions' => array(
		'active' => 1,
		'users.name' => 'john'
	),
	'add_tables' => array(
		array(
			'table' => 'comments',
			'conditions' => array(
				'active' => 1
			),
			'add_tables' => array(
				'table' => 'users',
			)
		),
		array(
			'table' => 'tags',
			'fields' => 'name'
		)
	)
));
