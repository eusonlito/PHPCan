The configuration of Content

The content module has one configuration file to manage which tables and folders can be managed by the module.

Tables configuration

The configuration of tables defines which tables, formats and relations will be managed by content module. By default all tables, formats and relations are availables.

$config['content_tables'] = $this->config['tables'];
$config['content_relations'] = $this->config['relations'];

All tables and relations are stored in $Config->tables and $Config->relations, so in this example we set the same values to $Config->content_tables and $Config->content_relations all tables will be managed.

$config['content_tables'] = $this->config['tables'];
$config['content_relations'] = $this->config['relations'];

unset(
	$config['content_tables']['default']['users'],
	$config['content_tables']['default']['comments']
);

In this example we can manage all tables but "users" and "comments" in "default" connection.

Rename and describe tables and formats

You can rename the tables and add a short description visible in content. This is helpful when the content manager is not the same than the website programmer. To do that, define the names and descriptions in $Config->content_tables_info.

$config['content_tables'] = $this->config['tables'];
$config['content_relations'] = $this->config['relations'];

$config['content_tables_info']['default'] = array(
	'categories' => array(
		'name' => 'The categories',
		'description' => 'Categories for the posts',

		'formats' => array(
			'name' => array(
				'name' => 'Category',
				'description' => 'Name of the category'
			),
			'text' => array(
				'name' => 'Description',
				'description' => 'Description of the category. They will be visible on mouse over'
			)
		)
	)
);

In this example the table "categories" is renamed to "The categories" and a description was added. Other names and descriptions where added to its formats "name" and "text".
